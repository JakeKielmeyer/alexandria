Page-Turn Conversion Plan: page-flip → Three.js / R3F
Context
The current flipbook uses page-flip v2.0.7, an HTML-canvas-based 2D engine that reparents DOM elements and renders a page-curl onto a <canvas>. The target stack replaces this with a WebGL/Three.js 3D scene (@react-three/fiber), types everything in TypeScript, swaps npm for pnpm, and adds Supabase, Sentry, framer-motion, react-router-dom, and html2canvas.

The single hardest problem: page-flip owns page content as live DOM nodes inside the canvas. In Three.js, page surfaces are GPU textures — not DOM. Every piece of "page content" must be converted to a texture source.

PageCurlMaterial.ts
A custom ShaderMaterial implementing the cylindrical fold deformation:

Vertex shader key logic:

uniform float uProgress;   // 0 = flat/unflipped, 1 = fully flipped
uniform float uPageWidth;
const float PI = 3.14159265;

void main() {
  vec3 pos = position;

  // The curl axis sits at the spine (left edge of this page = x = 0)
  // The free edge starts at x = uPageWidth and sweeps toward x = 0
  float freeEdge = uPageWidth * (1.0 - uProgress);

  if (pos.x > freeEdge) {
    float t = (pos.x - freeEdge) / max(uPageWidth * uProgress, 0.001);
    float angle = t * PI;
    // Cylindrical bend: project x inward, lift z outward
    pos.x = freeEdge - sin(angle) * (pos.x - freeEdge);
    pos.z += (1.0 - cos(angle)) * uPageWidth * uProgress * 0.3;
  }

  // Back face: flip normals so page has visible reverse side
  if (uProgress > 0.5) pos.x = -pos.x;

  gl_Position = projectionMatrix * modelViewMatrix * vec4(pos, 1.0);
}
Animate uProgress from 0 → 1 (flip to next) or 1 → 0 (flip back) using framer-motion's useSpring or R3F's useFrame + lerp. useSpring (spring physics) produces the natural deceleration of a real page; lerp in useFrame is simpler but linear.

Recommended: drive with useMotionValue + useSpring from framer-motion, and subscribe to the value in useFrame:

const progress = useMotionValue(0);
const spring = useSpring(progress, { stiffness: 80, damping: 18 });
useFrame(() => { material.uniforms.uProgress.value = spring.get(); });
PageMesh.tsx
One PlaneGeometry per page side (left/right of a spread):

<mesh>
  <planeGeometry args={[pageWidth, pageHeight, 64, 1]} />  {/* 64 x-segments for smooth curl */}
  <PageCurlMaterial
    map={pageTexture}        // texture of page content
    side={THREE.DoubleSide}  // visible from both sides during turn
    transparent
  />
</mesh>
Page content → texture (html2canvas integration): Each page has an off-screen <div> with the same content (parchment background, media, TextBlock) rendered into a hidden container outside the Canvas. html2canvas converts this div to a HTMLCanvasElement, which becomes a Three.js CanvasTexture.

const canvas = await html2canvas(hiddenDivRef.current, { useCORS: true });
const texture = new THREE.CanvasTexture(canvas);
For video content, skip html2canvas and use THREE.VideoTexture directly:

const texture = new THREE.VideoTexture(videoElement);
texture.minFilter = THREE.LinearFilter;
This replaces the entire videoRefs / syncVideos architecture. The video element is created in React, passed to the texture, and play/pause is controlled by the same isVisible logic as before — just against the raw <video> element instead of DOM nodes reparented by StPageFlip.

4. The full-spread-layer Architecture (Simplified)
The entire fullSpreadLayerRef / .is-hiding CSS class system is deleted. It existed only because StPageFlip reparented DOM elements.

Replacement: a single <mesh> in the R3F tree with PlaneGeometry(pageWidth*2, pageHeight).

mesh.visible = !isFlipping controls visibility
During a flip, PageMesh renders a static snapshot texture (same posterUrl logic)
After flip, mesh.visible = true and the video texture resumes playing
No DOM refs. No cancelable timers. No !important CSS.

5. framer-motion Role
framer-motion does not drive the page curl geometry directly (that's Three.js). It handles all 2D UI animations:

Animation	Implementation
Editor panel slide in/out	<motion.div> replace CSS right: 40% transition
Thumbnail panel slide up	<motion.div> replace CSS translateY(100%)
Toolbar button hover/press	whileHover, whileTap
Zoom scale transition	useSpring driving inner book wrapper scale
Book group X position (cover)	useSpring → useFrame → group.position.x
Page curl uProgress uniform	useSpring → useFrame → material.uniforms
Replace the existing transition: CSS strings with motion components throughout the UI layer.
